  • Structural cheat sheet:
-----------------------
.hero
  .wrapper
    .hero_card
      .hero_card-bg
        .hero_card-padding
  .hero_img-container.hero_img-container__small
    .hero_img
  .hero_img-container.hero_img-container__large
    .hero_img
  • .hero
  • For full screen edge to edge background color. We have chosen to set the background color using an inline style so that our templating system can dynamically set it. If it makes more sense for your project to set it with CSS then please do so.
  • .wrapper
  • To center the hero content with a max-width.
  • .hero_card-padding
  • This extra div is used to collapse the margins of child elements.
  • .hero_img-container
  • By using a wrapping element we can responsibly load a small or large screen background image by controling the display value of this element. More info on this approach can be found here: http://timkadlec.com/2012/04/media-query-asset-downloading-results/
  • .hero_img
  • The reason for using a background image is noted above in .hero_img-container. We have chosen to set the background image using an inline style so that our templating system can dynamically set the source. If it makes more sense for your project to set it with CSS then please do so.

Line container

  • .line-container is meant to contain anything you want with consistent top and bottom padding. It uses a margin collapsing trick in CSS to achieve this consistency which this is why .line-container_body is needed. The only caveat to this consistency is if you place something inside of it that has a bottom-margin that is larger than the .line-container padding.
